# 프로젝트: 술자리 밸런스게임

## 1. 서비스 개요

- **한 줄 설명:** 술자리가 더 재밌어지는 10초, 너의 선택은?
- **목적:** 술자리, MT 등 친구들과의 모임에서 로그인 없이 즉시 즐길 수 있는 간단하고 재미있는 밸런스게임 웹사이트 제공.
- **타겟 사용자:** 20-30대 소셜 모임을 즐기는 남녀. 설명을 읽지 않고, 즉각적인 반응을 원하며, 모바일에 익숙하다.

## 2. 디자인 및 UX 원칙

- **Mobile First:** 모든 디자인과 레이아웃은 모바일 화면에 최적화된다.
- **Minimal Clicks:** 사용자의 클릭 수를 최소화하여 즉각적인 게임 플레이를 유도한다. (랜딩 페이지 = 게임 시작 페이지)
- **One Screen, One Question:** 한 화면에는 하나의 질문만 노출하여 집중도를 높인다.
- **Intuitive Flow:** 선택 즉시 결과가 보이고, 자동으로 다음 질문으로 넘어가 막힘없는 흐름을 제공한다.
- **No Complex Navigation:** '뒤로 가기', '목록 보기' 등 불필요한 탐색 요소를 제거한다.

## 3. MVP 기능 명세

### Front-end
- **메인 화면:**
  - 3개의 게임 카테고리 선택 버튼 (`가벼운 맛`, `논쟁 유발`, `위험한 맛`)
  - 1개의 전체 랜덤 시작 버튼 (`아무거나 시작!`)
  - '오늘의 질문' 링크
- **게임 화면:**
  - 질문 텍스트 표시
  - 2-3개의 선택지 버튼 표시
  - 선택 시, 선택 비율(통계) 즉시 표시
  - 1.5초 후 다음 질문으로 자동 전환
  - 화면 전환 시 부드러운 애니메이션 효과
- **공유 기능:**
  - 현재 질문을 공유할 수 있는 아이콘
  - `/q/:id` 형태의 고유 URL 생성 및 클립보드 복사
- **라우팅:**
  - 사용자가 고유 URL로 접속 시, 해당 질문부터 게임을 시작하는 기능

### Back-end (Serverless Function)
- **API:**
  - `GET /questions?category={category}`: 카테고리별 랜덤 질문 목록 반환
  - `GET /questions/random`: 전체 랜덤 질문 반환
  - `GET /questions/today`: 오늘의 질문 반환
  - `GET /question/:id`: 특정 ID의 질문 반환
  - `POST /vote/:questionId/:choiceId`: 사용자의 선택을 기록하고 현재 통계 반환

### 데이터
- 20-30대 타겟에 맞는 오리지널 밸런스게임 질문 100개 이상
- 데이터 구조: `id`, `category`, `question`, `choices` 배열

## 4. 기술 스택

- **Core:** Vanilla HTML, CSS, JavaScript (ES Modules)
- **UI:** Web Components를 활용한 컴포넌트 기반 아키텍처
- **Infra:** 정적 웹 호스팅 (Firebase Hosting 등) 및 Serverless Functions (Firebase Functions 등)
